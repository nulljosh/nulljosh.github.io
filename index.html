001 · Finance Live E
Epiphany ↗
  • Problem: tracking markets, macro data, and portfolio performance meant juggling five different apps with no daily synthesis.

002 · Benefits Live T
Talli ↗
  • Problem: BC benefits (PWD, income assistance, MSP, 20+ programs) live behind a clunky government portal with no unified status view or deadline tracking.
  • Stack: Node/Express scraper API + Puppeteer against BC Self-Serve, vanilla JS PWA frontend, SwiftUI iOS + watchOS companions, Vercel Blob for persistence.
  • Decisions: session-encrypted credentials and a server-side scraper instead of an unofficial API, since BC Self-Serve has none — keeps user credentials off the client and survives portal changes more gracefully.
  • Outcome: live, in daily personal use for tracking a real PWD application and monthly reporting deadlines.

003 · Audio Live E
Echo ↗
  • Problem: cloud transcription tools require uploading audio to a third party — a non-starter for private recordings (interviews, voice memos, podcasts).
  • Stack: SwiftUI (iOS + macOS), WhisperKit running entirely on-device, no backend or network calls.
  • Decisions: chose on-device WhisperKit over a server API specifically to guarantee zero audio leaves the device — slower than a cloud GPU but the privacy guarantee was the actual point.
  • Outcome: 12-language transcription from mic or file import, fully offline.
